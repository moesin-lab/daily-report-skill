## 今日工作写法

### 板块

按工作类型聚合，最多 4 个板块：

- `### 修Bug`
- `### 新功能`
- `### 治理`
- `### 调研`
- `### 工具`

超过 4 类时，把命中少且认知增量为“无”的类型合并为 `### 其他`。

板块顺序按板块内最强 session 的认知增量降序排列。

### 单条结构

每条 session 使用三行结构：

```markdown
- **`#abc1234`** 事件一句话。
  - 认知：判断变化或新约束。
  - 残留：未验证、未合并、未修复或待跟进项。
```

规则：

- 事件、认知、残留各 ≤ 40 字。
- 无残留时省略残留行。
- 不把三行压成一段。
- 不展开迭代过程；必要时压进认知行，或留给第 2.4 步。

### 认知字段

认知只写判断变化或新约束发现。

不写：

- 完成事项。
- 工具使用事实。
- 可从 git log 直接还原的信息。

### `认知=无`

- `认知=无` 且 `残留=无`：压到板块末尾一行并列，格式为 `` `#X` A / `#Y` B：无增量``。
- `认知=无` 且 `残留≠无`：保留独立条目，省略认知行，保留残留行。
- `认知≠无`：按三行结构独立成段。

### 状态和合并卡片

- 状态直接写进事件或残留，不另起 tag。
- `session_id` 以 `merged-` 开头时，用 `merged-<group_id>` 作锚点，并说明合并自 N 个 session。
