## 沙盒外活动写法

独立板块 `## 沙盒外活动`，平行于 `## GitHub 活动`。

### 输入

`$OUTSIDE_NOTES_FILE`（来自第 1.5 步）。

### 触发条件

- `$OUTSIDE_NOTES_FILE` 为空或只含 `<!-- empty -->`：整节省略，不写空标题。
- 含有效 bullet：按原文顺序写入本节，不重新归并或排序。

### 单条结构

保留第 1.5 步的三行结构：

```markdown
- **HH:MM 事件标题** — 一句事件摘要。
  - 验证：...
  - 跟进：...
```

### 规则

- 本节不做认知/残留区分；沙盒外笔记已是外部事实，不套用 session 三段式。
- 事件摘要 ≤ 50 字，验证/跟进 ≤ 40 字，与第 1.5 步一致。
- 禁止把沙盒外笔记并入 `## 今日工作`，避免和 session 卡片体例混淆。
- 禁止把沙盒外笔记并入 `## GitHub 活动`，GitHub 活动只写 PR/Issue/Commit 编号。
- 多条笔记指向同一事件时，不做跨条归并；并列保留，读者自行识别。
