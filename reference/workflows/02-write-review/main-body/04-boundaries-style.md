## 总结 / 思考边界

总结只写客观状态：

- 今天整体推进到哪。
- 核心产出和工作重心变化。
- 2-3 句，≤ 150 字。

总结不写建议、反思或“下次应该”。

思考只写判断变化；完成事项和过程复述不进入思考。

## 排版

- 函数名、文件路径、PR 号、参数值、commit hash 用 inline `code`。
- 不全段加粗。
- 只加粗未验证、阻塞、残留风险等警示信息。

## 禁止写法

- 不加 TOC。
- 不在 `###` 后加 TL;DR 引用块。
- 不用表格替代演进型 bullet。
- 不用五段式展开过程。
- `<details>` 只用于审议过程附录。
